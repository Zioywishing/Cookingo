# AGENTS.md

按任务类型加载规则文件，不要无差别读取所有规则。

当前任务属于前端开发任务时，先阅读以下规则文件，并将其作为项目级约束严格遵守：

- `ai-coding/rules/frontend-rules.md`

当任务属于后端开发任务时，先阅读以下规则文件，并将其作为项目级约束严格遵守：

- `ai-coding/rules/server-rules.md`

前端开发任务包括但不限于：

- 新增或修改页面、布局、组件
- 新增或修改 composables
- 新增或修改前端插件、前端路由中间件
- 调整前端目录结构、文件归属、命名方式
- 定义或修改前端直接使用的共享类型
- 编写页面展示逻辑、交互逻辑、前端状态组织逻辑

后端开发任务包括但不限于：

- 新增或修改服务端接口
- 调整接口返回结构
- 定义或修改后端共享类型
- 设计或修改错误码
- 编写服务端业务逻辑、鉴权、参数校验或数据输出逻辑

执行要求：

- 仅在前端开发任务中读取并应用 `ai-coding/rules/frontend-rules.md`。
- 仅在后端开发任务中读取并应用 `ai-coding/rules/server-rules.md`。
- 若任务涉及页面、组件、composables、Nuxt 目录归属或前端共享类型，优先遵守前端规则文件。
- 若任务涉及后端接口、共享接口类型、错误码约定，优先遵守上述规则文件。
- 若本次改动与当前已沉淀的 rules 不一致，必须先与人二次确认；确认后，除执行本次改动外，还必须同步更新对应 rules，保持规则与实际实现一致。
- 不要修改与当前任务无关的文件。
- 若用户的明确要求与规则冲突，以用户要求为准。
